本文档为**方法论进化器**小程序最终定稿需求文档，固化所有产品逻辑、数据结构、功能规则、交互标准，作为唯一开发依据，全程无冗余功能、无逻辑变更预留，严格贴合个人专属精进工具定位。

## 1\.3 最终产品定义（用户定稿）

**一个帮助我个人进步的方法论进化器，旨在提升我做各类事情的成功率**。

产品本质：纯私人、主观驱动、持续迭代的个人做事方法论沉淀工具，无社交、无推荐、无公共内容，仅服务单一用户。

# 二、产品顶层核心公理（永久不可修改）

本次产品全部功能基于以下8条公理搭建，为产品底层灵魂：

1. **绝对主观确权**：所有规律、对的事均由用户主观录入判定，系统不做任何客观对错评判，仅做收纳、计算、关联、迭代载体。

2. **唯一终极目标**：一切记录行为只为沉淀、进化专属个人方法论，最终提升用户各类事件做事成功率，非打卡、流水、日记工具。

3. **二维分类体系**：表层为场景分类（职场、运动、股票等），底层为逻辑分类，服务跨场景迁移能力。

4. **规律可迁移**：打破场景壁垒，实现底层通用逻辑跨行业、跨领域复用，拔高做事通用性能力。

5. **规律二元性**：规律无对错，分为正向规律、负向规律；对的事永远为正向优质动作，负向规律禁止归入对的事，用于避雷纠错。

6. **动作量化原则**：所有对的事可量化统计执行次数、成功次数、成功率，支持多维排名、数据复盘。

7. **沉淀形态优先级**：碎片化卡片 ＞ SOP执行模板 ＞ 结构化手册，三者共存，按需切换使用。

8. **迁移双模并存**：同时支持用户手动自主迁移、系统智能逻辑推荐迁移，可自主开关智能提醒。

# 三、用户最终确认选型（固化执行）

第三轮讨论4项关键选型，全部定稿，严格落地：

1. **执行结果判定**：1A 纯主观判定，由用户手动勾选单次执行成功/失败，不受客观数据干预。

2. **负向规律功能**：2B 存档记录\+避雷提醒，同类事件触发时弹窗提示过往负向坏规律，规避错误惯性。

3. **智能迁移逻辑**：3B 逻辑句式匹配，优先识别因果底层逻辑，实现跨大类场景迁移推荐（如股票低吸逻辑迁移实体进货）。

4. **安全存储权限**：4A 本地存储\+云端双重备份，无任何对外分享入口，极致私密。

# 四、核心业务四层数据结构（代码级底层架构）

为产品核心骨架，所有数据严格分层、互不冗余，以股票低吸高抛为案例释义：

## 4\.1 第一层：原始事件层（流水原始记录）

记录单次行为原始信息，无加工、无总结、无定性。包含字段：记录时间、场景分类、事件描述、主观备注、单次情绪/状态。示例：2026\.05\.07，理财\-股票，买入XX股票，低吸操作，当日盈利。

## 4\.2 第二层：动作层（对的事，纯正向）

用户主观判定优质、需要重复执行的正向动作，无负面动作录入。包含字段：动作名称、所属分类、创建时间、累计执行次数、成功次数、失败次数、成功率、主观权重、关联标签。示例：理财\-低吸高抛。

## 4\.3 第三层：规律层（二元拆分，产品核心亮点）

### 4\.3\.1 正向规律

从多次成功动作中提炼因果逻辑，用于复用强化。示例：行情震荡期，低吸高抛容错率更高。

### 4\.3\.2 负向规律

从失败行为、错误思维中总结坏逻辑，用于避雷规避，不纳入对的事。示例：情绪化追涨，大概率短期被套。

## 4\.4 第四层：方法论层（最终产出物）

### 4\.4\.1 碎片化卡片（最高优先级）

单条动作、单条规律独立生成卡片，自由查看、编辑、收藏、关联，轻量化管理。

### 4\.4\.2 SOP执行模板（次优先级）

多条动作\+正向规律组合，生成标准化做事流程，直接套用。示例：股票交易SOP=判断行情→低位建仓→设定止盈→高位卖出。

### 4\.4\.3 结构化手册（最低优先级）

系统自动汇总同分类逻辑，生成私人方法论手册，长期沉淀归档。

# 五、核心功能清单（全部功能，无多余附加）

## 5\.1 录入模块（基础核心）

1. 事件录入：自由记录单次原始事件，支持自定义备注；

2. 对的事录入：手动新增正向动作，绑定分类、初始权重；

3. 规律录入：手动区分正向/负向规律，绑定关联动作、所属场景；

4. 执行打卡：对已录入的对的事，一键新增执行记录，手动勾选成功/失败，自动更新统计数据。

## 5\.2 数据量化模块（量化成功率）

1. 基础统计：自动计算执行次数、成功数、失败数、成功率；

2. 多维排名：支持按执行次数、成功率、主观权重排序，生成个人动作榜单；

3. 数据趋势：生成周期曲线图，查看单条动作成功率变化、执行频率变化。

## 5\.3 迁移模块（产品高阶能力）

1. 手动迁移：用户自主将某分类的规律/动作，复制关联至其他分类；

2. 智能迁移：系统识别逻辑句式，跨场景匹配同类底层规律，弹窗推荐迁移；

3. 迁移记录：留存所有迁移痕迹，查看逻辑复用历史。

## 5\.4 复盘迭代模块（方法论进化核心）

1. 周期复盘：支持自定义复盘周期（日/周/月），自动生成复盘报告；

2. 迭代操作：复盘内可对动作/规律执行固化、修改、降级、淘汰操作；

3. 负向提醒：执行同类事件时，弹窗展示关联负向规律，规避错误；

4. 版本留存：每次迭代留存历史版本，查看方法论进化轨迹。

## 5\.5 方法论沉淀模块（最终产物）

1. 卡片库：全部动作、规律以极简卡片展示，支持检索、筛选、收藏；

2. SOP模板库：手动\+自动组合生成做事流程模板，一键套用；

3. 私人手册：自动汇总分类内容，生成可查看的结构化方法论文档。

## 5\.6 安全存储模块

1. 双重存储：本地缓存\+云端加密备份，防止数据丢失；

2. 零分享入口：无任何分享、转发、社交功能，绝对私密；

3. 数据管理：支持手动删除、归档、加密隐藏敏感记录。

# 十、前端页面\&amp;交互设计（开发定稿）

本章为前端开发唯一依据，包含页面架构、页面清单、布局、交互手势、弹窗规则、UI视觉、页面跳转，无自由发挥空间，严格强制执行。

## 10\.1 整体产品架构

采用小程序标准底部Tabbar架构，**5个一级页面，最大嵌套层级2级**，杜绝深层跳转，追求极致操作效率。一级页面：首页工作台、卡片库、SOP模板、复盘中心、个人中心。

## 10\.2 全局硬性交互规则（全页面通用）

### 10\.2\.1 手势交互

1. 单击：查看详情、打开弹窗；

2. 长按：任意卡片唤起操作菜单（编辑/迁移/归档/删除）；

3. 右滑：列表卡片快速打卡（成功/失败）；

4. 下拉：任意页面下拉呼出全局快捷录入悬浮按钮；

5. 右滑返回：所有二级页面支持原生手势返回。

### 10\.2\.2 弹窗规则（严格限制弹窗频次）

1. 负向避雷弹窗：同分类单日最多弹出2次，可手动永久关闭单条规律提醒；

2. 智能迁移弹窗：每日最大推送3条，无强制拦截，点击空白关闭；

3. 高危操作弹窗：删除、清空、永久归档必须二次弹窗确认，标注风险红字；

4. 所有弹窗无自动轮播、无强制停留。

### 10\.2\.3 草稿保存规则

1. 所有输入框实时自动存草稿，退出页面不丢失；

2. 未提交草稿在首页顶部展示「未完成录入」提醒；

3. 草稿存储方案：**本地 storage（uni.setStorageSync）**，无需服务端接口，清除缓存或换设备后草稿不保留，属于预期行为。

## 10\.3 全局UI视觉规范（设计师\&amp;前端统一标准）

### 10\.3\.1 色彩体系

1. 主色：深墨蓝 \#162238（理性、专注、商务）；

2. 辅助色：浅灰 \#F5F7FA、纯白 \#FFFFFF；

3. 状态色：成功绿 \#36D399、失败红 \#F87272、中性灰 \#9CA3AF；

4. 禁止使用：高饱和亮色、渐变花哨配色。

### 10\.3\.2 字体规范

1. 字体：微信默认无衬线字体；

2. 标题：16px 加粗；正文：14px 常规；备注：12px 浅灰；

3. 行间距：1\.5倍，保证长文本可读性。

### 10\.3\.3 卡片规范

1. 卡片圆角：12rpx；卡片阴影：轻度悬浮阴影；

2. 卡片底色：纯白/深色模式深灰；

3. 动效：仅弹窗渐变、卡片悬浮轻微放大，无多余转场动画。

### 10\.3\.4 模式适配

支持**三档模式切换**：浅色模式、深色模式、跟随系统。默认跟随系统，可在个人中心手动固定为浅色或深色。数据库存储值：0=浅色、1=深色、2=跟随系统。

## 10\.4 一级页面详细开发说明

### 10\.4\.1 首页\-工作台

页面定位：数据总览\+快捷入口，用户第一视觉页。

页面固定模块（不可删减顺序）：

**模块一：数据看板**

三列均分展示，分别对应三个核心起点：

| 正确的事 | 正向规律 | 负向规律 |
|--------|--------|--------|
| 总数 N 条 | 总数 N 条 | 总数 N 条 |
| 触发 N 次 | 触发 N 次 | 触发 N 次 |

- 正确的事触发次数 = 累计打卡执行次数
- 正向规律触发次数 = 被关联引用/SOP执行次数
- 负向规律触发次数 = 避雷弹窗被触发次数
- 每格点击跳转对应详情页（TODO：H5阶段暂不实现，点击无反应）

**模块二：快捷操作区**

一行四格，固定顺序：

| 新增正确的事 | 新增正向规律 | 新增负向规律 | 打卡 |

- 前三个按钮各自打开对应录入弹窗
- 打卡按钮打开打卡弹窗，流程：选择正确的事 → 选择成功/失败 → 填写备注（选填）→ 确认打卡
- 打卡仅针对"正确的事"，规律不参与打卡
- 打卡弹窗点击空白处关闭，本次打卡作废不保存

**模块三：待办提醒区**

每种待办独立一张卡片，垂直堆叠，全部无数据时整区隐藏：

1. 待复盘提醒：有未完成复盘时显示「您有N个正确的事待复盘」，点击跳转复盘中心
2. 迁移推荐提醒：每日打开App时后台计算，全局最多3条，显示「今日有N条迁移推荐」，点击跳转卡片库
3. 长期未执行提醒：超过7天未打卡的正确的事，每条独立显示「XXX 已N天未执行」，点击跳转该卡片详情
4. 所有待办卡片右侧有「×」可手动清除，清除后当日不再显示

**模块四：我的榜单**

左右滑动切换三个榜单，默认展示正确的事：

- 正确的事榜单：按主观权重降序，展示名称、分类、执行次数、成功率
- 正向规律榜单：按主观权重降序，展示名称、分类、触发次数
- 负向规律榜单：按主观权重降序，展示名称、分类、触发次数
- 顶部小圆点指示当前位置（●○○）
- 右上角「自定义」按钮，可分别设置三个榜单显示条数
- 榜单每行点击跳转对应卡片详情（TODO：H5阶段暂不实现）

**概念统一说明（全局生效）：**

全项目用户可见文案统一使用"正确的事"，不使用"动作"。代码变量名（action/actions）保持不变，仅文案层替换。

### 10\.4\.2 卡片库页面

页面定位：全量碎片化数据收纳，检索管理。

顶部筛选栏：支持多选筛选（场景分类\+逻辑标签\+成功率区间\+时间）；

### 10\.4\.3 SOP模板页面

页面定位：流程化做事模板。

模块：我的模板、智能推荐模板、新建模板；

## 10\.5 二级核心页面（弹窗）开发规则

### 10\.5\.1 录入页面（事件/对的事/规律）

1. 事件录入：时间、分类、事件描述、主观情绪、备注；

2. 对的事录入：名称、分类、主观权重、初始备注；

3. 规律录入：**强制单选正向/负向**、关联动作、因果描述、适用场景；

4. 非空校验：名称不可为空，空内容禁止提交。

### 10\.5\.2 快速打卡弹窗

1. 固定双按钮：成功、失败；

2. 附带单行备注框，记录本次执行细节；

3. 提交后实时刷新该动作统计数据，无需手动刷新页面。

### 10\.5\.3 迁移操作页面

1. 手动迁移：勾选目标分类，复制生成新卡片，原卡片保留；

2. 智能迁移：展示逻辑相似度百分比，标注迁移来源；

3. 迁移记录永久留存，不可删除。

# 十一、数据库表结构（后端开发级，不可修改字段）

所有数据表严格遵循四层数据结构，字段齐全、索引明确，适配MySQL。

## 11\.1 用户主表（user）

## 11\.2 原始事件表（event\_raw）

# 十二、业务硬性判定逻辑（后端写死，无自定义）

## 12\.1 成功率计算规则

成功率 = 成功次数 /（成功次数\+失败次数）×100%；无执行记录时显示「暂无数据」，禁止出现0%误导。

## 12\.2 智能迁移匹配算法规则

采用**语义句式匹配**，不匹配关键词；提取因果逻辑主谓结构，逻辑相似度＞65%判定为可迁移，弹窗推荐。

> 决策说明：智能迁移推荐**不在录入时实时触发**，改为用户每次打开 App 时后台静默计算，结果展示在首页待办提醒区"迁移推荐"模块。全局每日最多推送3条，跨分类合并计算，不按分类单独限额。用户可对每条推荐选择「采纳、忽略、永久关闭该推荐」。

## 12\.3 负向提醒触发规则

新增事件/动作时，匹配同分类下负向规律，触发弹窗提醒；单条规律24小时内仅提醒1次。

## 12\.4 数据排序权重规则

支持三种排序：执行次数降序、成功率降序、主观权重降序。

## 12\.5 迭代状态规则
