# Feature Alignment: H5 与需求文档对齐

## 背景

H5 项目作为小程序的预览版本，经过 UI 重设计后视觉层面已基本完成。但对照需求文档，存在若干功能缺口需要补齐。本 proposal 按优先级排列需要处理的不一致项。

## 变更范围

按优先级顺序实施以下 4 项功能对齐：

### Task 1: 复盘迭代操作（核心缺口）

**现状**：复盘中心仅展示历史版本和可视化图表，不支持"新建复盘"操作。
**文档要求**：用户可选择复盘周期（周/月）→ 系统自动拉取该周期数据 → 逐条对现有方法论进行判定（固化/优化/降级/淘汰）→ 生成新版本快照。
**实施内容**：
- 复盘中心顶部增加"新建复盘"入口
- 新建复盘流程：选周期 → 展示周期内数据 → 逐条判定 → 提交生成快照
- Mock API 补充：`createReview`、`submitIteration`

### Task 2: 负向规律避雷弹窗（创建时触发）

**现状**：打卡时已接入 `checkNegativeLaws`，但创建"正确的事"或"错误的事"时未检测同分类负向规律。
**文档要求**：录入新动作时自动检测同分类负向规律，弹窗提醒用户注意（24小时去重）。
**实施内容**：
- CreateActionModal 提交前调用 `checkNegativeLaws(categoryId)`
- 检测到匹配时弹出避雷提醒弹窗
- 用户可选择"知道了，继续"或"取消"
- 复用已有的 `checkNegativeLaws` Mock 逻辑

### Task 3: 灵感转化流程完善

**现状**：卡片库中灵感长按菜单已有"转为正确的事/错误的事/规律"选项，但点击后直接调用 `convertInspiration` API 并打开编辑弹窗。流程基本完整。
**需确认**：转化后是否正确预填数据到编辑弹窗？当前代码逻辑看起来已实现，需验证是否正常工作。
**实施内容**：
- 验证灵感转化流程是否正常
- 修复可能的预填数据丢失问题

### Task 4: 数据看板点击跳转

**现状**：首页数据看板（正确的事/错误的事/正向规律/负向规律）的数字不可点击。
**文档要求**：点击数字跳转到卡片库对应 Tab。
**实施内容**：
- DataDashboard 组件各数字项加 onClick
- 点击后调用 `onSwitchTab('card')` 并传递目标 subTab

## 不在本次范围

- 备份还原（需后端支持）
- 隐私设置/密码锁（第三优先级）
- 事件原始流水表（v2 延期）
- SOP 拖拽排序（箭头方案已满足需求）
- 榜单点击跳转（低优先级）

## 验证方式

每个 Task 完成后：
1. dev server 刷新验证功能正常
2. 确认不影响已有功能
3. build 通过无报错
