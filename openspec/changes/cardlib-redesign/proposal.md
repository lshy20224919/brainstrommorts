## Why

卡片库页面是用户日常使用频率最高的页面，但当前设计存在三个结构性问题：

1. **分类逻辑扁平**：正确的事、错误的事、规律、灵感四个 Tab 平铺，没有体现它们之间的认知层级关系（行为→规律→灵感）
2. **卡片视觉不统一**：四种卡片各自为政，信息排布、交互方式完全不同，用户在 Tab 间切换时没有一致的阅读节奏
3. **交互方式混乱**：ActionCard 有滑动手势+长按+箭头，其他卡片只有长按，H5 上滑动与页面滚动冲突且发现率低

## What Changes

### 信息架构重构

一级 Tab 改为三类认知形态：

| 一级 | 二级 | 数据来源 |
|------|------|----------|
| 行为 | 正确的事 | actions |
| 行为 | 错误的事 | mistakes |
| 规律 | 正向规律 | laws (law_type=1) |
| 规律 | 负向规律 | laws (law_type=2) |
| 灵感 | 正向灵感 | inspirations (direction=positive) |
| 灵感 | 负向灵感 | inspirations (direction=negative) |

### 卡片视觉统一

所有卡片共享三行骨架：标题行 → 元信息行 → 数据行，各自填充不同内容。

### 交互模型统一

- 点击卡片 → 进入详情
- 长按卡片 → 操作菜单
- 去掉滑动手势，打卡入口移至详情页内

### 筛选栏统一

所有 Tab 统一为 [分类▾] [排序▾] 两个筛选器，排序选项因 Tab 而异。去掉状态筛选器，归档内容默认隐藏。

### 数据模型变更

灵感新增 `direction` 字段（positive/negative），创建时选择。

## Non-goals

- 不改变各卡片的业务逻辑和数据流
- 不重构详情页和弹窗组件
- 不涉及首页和复盘页的改动
