## ADDED Requirements

### Requirement: 正确的事卡片左划查看详情
系统 SHALL 允许用户对“正确的事”卡片执行左划手势，并打开该动作的详情面板。规律卡片 MUST NOT 支持该左划详情交互。

#### Scenario: 左划正确的事卡片
- **WHEN** 用户在“正确的事”卡片上执行左划，且位移超过左划阈值
- **THEN** 打开该动作的详情面板

#### Scenario: 规律卡片不支持左划详情
- **WHEN** 用户在规律卡片上执行左划
- **THEN** 不打开详情面板，规律卡片保持原有交互行为

### Requirement: 卡片左划提示 UI
系统 SHALL 在“正确的事”卡片上显示左划功能提示，包括右侧常驻箭头和浅灰辅助文案“左划查看详情”。

#### Scenario: 正常展示提示
- **WHEN** 用户进入卡片库并查看“正确的事”卡片
- **THEN** 每张卡片右侧显示小箭头，底部显示浅灰提示文案“左划查看详情”

### Requirement: 详情面板使用全屏右侧滑入
系统 SHALL 使用全屏右侧滑入的详情面板展示动作详情，而不是底部弹窗。

#### Scenario: 打开详情面板
- **WHEN** 用户左划动作卡片成功
- **THEN** 详情面板以从右向左滑入的动画全屏展示

#### Scenario: 关闭详情面板
- **WHEN** 用户点击返回按钮或执行关闭手势
- **THEN** 详情面板向右滑出并关闭

### Requirement: 详情展示动作概览
详情面板 SHALL 展示该动作的基础概览信息：名称、分类、执行次数、成功次数、失败次数、成功率、主观权重。

#### Scenario: 查看动作概览
- **WHEN** 用户打开某动作的详情面板
- **THEN** 顶部概览区显示该动作的名称、分类、执行次数、成功次数、失败次数、成功率和主观权重

### Requirement: 详情展示打卡记录时间线
详情面板 SHALL 展示该动作的打卡记录时间线，按执行时间倒序排序。每条记录展示时间、成功/失败状态、备注。

#### Scenario: 查看打卡记录
- **WHEN** 用户打开某动作详情面板，且该动作存在多条打卡记录
- **THEN** 打卡记录按时间倒序排列，每条显示执行时间、成功/失败和备注

#### Scenario: 无打卡记录
- **WHEN** 用户打开某动作详情面板，且该动作暂无打卡记录
- **THEN** 记录区显示空状态提示，如“暂无打卡记录”

### Requirement: 详情展示关联规律
详情面板 SHALL 展示与该动作关联的规律列表，包含正向规律和负向规律。

#### Scenario: 存在关联规律
- **WHEN** 某动作存在 `related_action_id = action.id` 的规律
- **THEN** 详情页展示这些规律，并标注正向/负向类型

#### Scenario: 无关联规律
- **WHEN** 某动作没有任何关联规律
- **THEN** 关联规律区显示空状态提示

### Requirement: 单条动作记录查询 API
系统 SHALL 提供按动作 ID 获取打卡记录的 API，并按时间倒序返回。

#### Scenario: 查询动作记录
- **WHEN** 详情面板请求某动作的打卡记录
- **THEN** API 返回该动作的全部记录，并按 exec_time 从新到旧排序
